#今後の案として以下のことを考えている。
##新規登録時にスマホのカメラで書類を撮影して、それを新規案件の中に取り込む。保存先はgoogleドライブ。共有者は案件データと写真で確認し作業にあたれる。




原因が分かりました。GASプロジェクトのマニフェストファイル (appsscript.json) にDrive権限のスコープが宣言されていないためです。

src/gas/appsscript.json を作成しました。以下の手順で設定してください：

手順
GASエディタを開く
左側メニューの 歯車アイコン（プロジェクトの設定） をクリック
「appsscript.json」マニフェストを表示する にチェックを入れる
左側メニューに appsscript.json が表示されるのでクリック
内容を以下に完全に置き換える：
json
{
  "timeZone": "Asia/Tokyo",
  "dependencies": {},
  "exceptionLogging": "STACKDRIVER",
  "runtimeVersion": "V8",
  "webapp": {
    "executeAs": "USER_DEPLOYING",
    "access": "ANYONE_ANONYMOUS"
  },
  "oauthScopes": [
    "https://www.googleapis.com/auth/spreadsheets",
    "https://www.googleapis.com/auth/drive"
  ]
}
Ctrl+S で保存
新しいデプロイを作成（「デプロイ」→「新しいデプロイ」）
新しいURLをスマホで開く
これでDrive権限が正しく要求されるはずです。
