# 実装計画：UI改善（静的モックアップ）

## 1. 目的
GASのデプロイを待たずにUIデザインを高速に反復開発するため、ローカルブラウザで直接動作するUIモックアップファイルを作成する。

## 2. 実装方針
- ルートディレクトリに `index-ui.html` を作成する。
- 既存の `src/gas/Index.html` の構造をベースにする。
- GAS固有の API コール (`google.script.run`) を JavaScript の `Promise` や `setTimeout` を使用した擬似的な非同期通信に置き換える。
- `Vue.js` のデータ初期値に実用的なサンプルデータをセットする。
- 開発バージョンタグを付与し、キャッシュ問題を回避する。

## 3. 変更内容
### `index-ui.html`
- **カードのコンパクト化**:
    - 上下パディングを 18px から 8px へ削減。
    - 住所・号室・地区を1行にまとめ、フォントサイズを 1.15rem 程度に最適化。
    - 鍵情報（玄関新/古、物置、PS）を個別のボックスではなく、アイコン付きのインライン要素として横並びに配置。
    - 日付と担当者を最下部の1行に集約し、マージンを最小化。
- `google.script.run` を `MockGas` クラス等でラップし、正常系・異常系の動作をシミュレート可能にする。
- デザインは premium なモダンデザインを目指す。

## 4. スケジュール
1. 基盤作成：既存コードの移植とモック化
2. デザイン改良：ビジュアル要素の強化
3. ユーザー確認：見た目のみの確認
4. 本番反映：`src/gas/Index.html` への統合

## 5. リスク・留意点
- モックデータと本番データの構造不一致。
- 外部ライブラリ（Bootstrap, Vue.js）のCDN依存。
